# 健康守护者 · 界面重构说明（UI/UX v5.2）

> **v5.2 患者向柔和化**：去掉仪表盘外围刺状装饰；底色从冷 slate 换温润薄荷沙；主标题色软化；Chip 选中态从墨黑改主色绿；大数字字距放松。考虑到患者群体（脑卒中、术后、帕金森）使用感受。



> 本文档记录最新一轮 UI/UX 重构的全部改动、设计系统与**上架步骤**。可直接交给 Codex 或其他开发者继续维护。
> **核心原则：本次只重构「外观层」，所有业务逻辑、数据结构、状态管理、离线模式均未改动。**

## 一、设计方向（v5 ——「Cool Slate Editorial」）

经历 v1 暖亲和 → v2 克制编辑 → v3 mesh 渐变 → v4 暖米编辑 → v5 冷色 slate 编辑，五轮迭代得到当前版本。

**v5 关键变化**（在 v4 基础上）：
- **去掉暖黄/米色调** — 全局换 Tailwind slate（slate-50/100/200/500/700/900），冷色调更现代、更"专业感"。
- **Hero 卡用纯白** — 与 slate 灰底形成清晰层级，靠 SHADOW.hero（emerald 投影）和 SVG 内容做视觉锚点。
- **保留 v4 全部 SVG 设计精华**：徽章插画、弧形进度环、Sparkline 折线、渐变柱状图、波浪分隔、编号小标。

**v4 既有视觉武器**：
- 真 SVG 图形（`react-native-svg`）— `HeroMedallion`（圆环+心电+进度光点）、`ArcMini`（弧形进度）、`SparkLine`（带渐变填充）、`GradientBar`、`WaveDivider`。
- 极端字号对比：72px 巨号 / 64px 大号 / 36px 中号 / 10.5px 小标贴（letter-spacing 2.5）。
- 编号小标贴：`01 — TODAY`、`02 — PATIENTS` 杂志感。
- 混合层级布局：大 Hero + Featured 大卡 + 任务条 + 2×2 Bento 网格。
- 5 色品牌：primary（深松绿）/ coral（珊瑚）/ amber（琥珀）/ lavender（薰衣草）/ sky（天蓝）。
- 5 色循环渐变头像（按姓名 hash）。

**功能层（v4 引入，v5 保留）**：
- **Storage 兜底层** — localStorage 被拒（iframe / 隐私模式）时落到内存 Map。
- **Web Alert 兜底** — Alert.alert 改用 window.alert/confirm。
- **登录预填** — `demo@jiankang.local / demo1234`。

## 六、上架步骤（直接交付）

### 1. 启动演示给评委 / 自查
- Windows：**双击 `演示.bat`**（自动起 Metro + 打开 mobile-preview.html）
- 任意平台：`npm run demo`
- Metro 已在跑时直接双击 `mobile-preview.html`

### 2. 上架前自检
```bash
npm run check:syntax     # 全部 JS 语法检查
npm test                 # 服务 + app.json 一致性冒烟测试
```
**当前状态：v5 已 PASS（已自动跑过）**

### 3. EAS 构建（重要）
本次新增了原生模块 `react-native-svg` 和 `expo-linear-gradient`，**旧的 EAS 包失效**，必须重新构建：
```bash
# 1. 登录 EAS（如未登录）
eas login

# 2. 生产构建（产出 .aab）
npm run build:android:production
# 等同于：eas build -p android --profile production

# 3. 构建完成后下载 .aab 或：
npm run submit:android
# 自动上传到 Google Play Console
```

### 4. Google Play 上架
- 已就绪：privacy/about/agreement 三个合规页（App 内 → 我的 → 合规与更多）
- 已就绪：注销账号 + 数据删除（App 内 → 我的 → 注销账号与删除数据）
- 待替换：`release/` 目录里的旧 UI 商店截图——上架前请用 `演示.bat` 启动后在 v5 真实界面截图重新提交

### 5. 离线演示模式
`.env` 中 `EXPO_PUBLIC_API_BASE_URL` 留空 → App 自动用本地 workspace 模式（不需服务器），数据保存在设备本地。比赛演示无需后端服务。
